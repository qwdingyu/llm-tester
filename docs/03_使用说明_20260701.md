# LLM Tester 使用说明

> 版本: v0.1.0 | 更新: 2026-07-01

## 一、项目概述

LLM Tester 是一个轻量级 LLM API 测试桌面工具，支持多种 API 格式的统一测试和基准对比。

**适用场景：**
- 快速验证 API Key 和模型是否可用
- 对比不同模型的响应质量和速度
- 压力测试模型的并发和 Token 消耗
- 批量测试多个 API 端点的连通性

**支持 Provider：** OpenAI 兼容 / Ollama / Azure / 自定义端点

## 二、快速开始

### 2.1 下载与启动

```bash
# 从 Release 页面下载对应平台的二进制后：
# macOS / Linux
./llm-tester

# Windows
llm-tester.exe

# 或自行编译
go build ./cmd/llm-tester/ && ./llm-tester
```

启动后打开浏览器访问: **http://localhost:8912**

> 端口可通过环境变量 `PORT` 修改，如 `PORT=9090 ./llm-tester`

### 2.2 使用流程

```
1. 配置管理 → 选择预设或手动填写 API 配置 → 保存
2. 连接测试 → 选择配置 → 测试连接（验证连通性和模型列表）
3. 聊天测试 → 选择配置 → 输入消息 → 发送
4. 批量测试 → 选择多个配置 → 输入消息 → 运行
5. 基准测试 → 选择配置 → 设置参数 → 运行（获取 P50/P95 延迟）
6. 压力测试 → 选择配置 → 设置轮次 → 运行（追踪 Token 消耗）
```

## 三、配置管理

### 3.1 配置字段说明

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 配置名称 | ✅ | 唯一标识，用于在其他标签页选择 |
| API 类型 | ✅ | openai / ollama / azure / custom |
| API 地址 | ✅ | Base URL，**不包含 `/v1`**（代码自动添加） |
| API Key | ✅ | 认证密钥，支持显示/隐藏切换 |
| 模型名称 | ✅ | 如 gpt-4o-mini、deepseek-chat |
| 端点模式 | 可选 | `chat_completions`（默认）或 `responses` |
| 温度 | 可选 | 0-2，默认 0.7 |
| Max Tokens | 可选 | 最大输出 Token 数 |
| 超时(秒) | 可选 | 0=默认 60s |
| 代理地址 | 可选 | 如 `http://127.0.0.1:7890` |
| 自定义路径 | 可选 | 仅 custom 类型使用 |
| HTTP-Referer | 可选 | 部分 API 需要 |
| X-Title | 可选 | 部分 API 需要 |

### 3.2 预设模板

内置 8 个预设模板，点击「预设」按钮选择：

| 预设 | API 类型 | 地址 |
|------|---------|------|
| OpenAI | openai | https://api.openai.com |
| DeepSeek | openai | https://api.deepseek.com |
| Kimi | openai | https://api.moonshot.cn |
| GLM-4 | openai | https://open.bigmodel.cn/api/paas/v4 |
| 通义千问 | openai | https://dashscope.aliyuncs.com/compatible-mode/v1 |
| Ollama | ollama | http://localhost:11434 |
| Azure | azure | https://YOUR_RESOURCE.openai.azure.com |
| Custom | custom | 手动填写 |

> **注意：** 预设仅填充 URL 和模型名，API Key 需要手动填写。

### 3.3 导入/导出

- **导出：** 所有配置导出为 JSON 文件下载
- **导入：** 支持单个对象 `{"name":"...", "base_url":"...", ...}` 或数组 `[{...}, {...}]`

### 3.4 配置存储位置

配置文件保存在 `~/.llm_tester/configs.json`（仅当前用户可读写）。

## 四、连接测试

选择已保存的配置，点击「测试连接」。

- 成功：显示可用模型列表
- 失败：显示具体错误原因（API Key 无效 / 连接超时 / 端点不存在 等）

**常见连接失败原因：**

| 错误信息 | 原因 | 解决 |
|---------|------|------|
| API Key 无效 | API Key 错误或已过期 | 检查 API Key |
| 额度已用尽 | 账户余额不足 | 充值或更换 Key |
| 连接超时 | 网络不通或代理未配置 | 配置代理地址或检查网络 |
| 端点不存在 | API 地址路径错误 | 确认 base_url 是否正确（不含 `/v1`） |

## 五、聊天测试

发送单条消息测试模型回复质量和速度。

**参数：**
- 配置：选择已保存的配置
- 温度：控制随机性（0=确定，2=随机）
- Max Tokens：最大回复长度
- 超时(秒)：请求超时时间
- 消息：输入要发送的内容

**结果展示：**
- 回复内容（等宽字体代码块）
- 延迟（ms）
- Token 统计（Prompt / Completion / Total）
- Finish Reason（stop=正常结束，length=截断）

> 提示：按 `Enter` 快捷键发送，支持「清空消息」「清空结果」按钮。

## 六、批量测试

同时测试多个配置的响应速度和可用性。

**流程：**
1. 点击选择要测试的配置（多选）
2. 输入测试消息
3. 点击「运行批量测试」

**结果：** SSE 实时推送每个配置的测试结果，含延迟和状态。

## 七、基准测试

对单个配置进行多轮并发测试，获取详细延迟统计。

**参数：**
- 轮次：测试总次数
- 并发数：同时发起的请求数
- 测试提示词：自定义测试消息
- Max Tokens / 超时

**统计报告指标：**

| 指标 | 说明 |
|------|------|
| 总请求 | 计划执行的请求总数 |
| 成功/失败 | 请求成功/失败数量 |
| 平均延迟 | 所有成功请求的平均耗时 |
| 最小/最大延迟 | 最快和最慢的请求 |
| P50 | 中位数延迟 |
| P95 | 95% 请求的延迟上限（排除长尾） |
| 标准差 | 延迟的离散程度（越小越稳定） |

## 八、压力测试

对单个配置进行长时间连续测试，追踪总 Token 消耗。

**参数：**
- 测试轮次：连续测试次数
- Max Tokens / 超时

**结果：** 总请求数、成功率、总 Token 消耗、平均 Token/轮。

## 九、日志面板

右侧日志面板实时显示所有操作记录（自动 2 秒轮询刷新）。

- 不同颜色区分成功/失败/信息
- 最多保留 500 条记录
- 支持「清空」按钮

## 十、代理配置

两种方式配置 HTTP 代理：

1. **配置级别（推荐）：** 在配置表单中填写「代理地址」
2. **环境变量（兜底）：** 设置 `HTTP_PROXY` / `HTTPS_PROXY`

配置级别的代理优先于环境变量。

## 十一、常见问题

### Q: 连接测试成功但聊天测试超时？

**原因：** API 返回了 SSE 流式响应。部分 OpenAI 兼容 API 在不指定 `stream` 参数时默认走流式输出。

**解决：** 此问题已在 v0.1.0 中修复，所有请求都显式添加 `"stream": false`。

### Q: base_url 是否包含 /v1？

**不需要。** 代码会自动添加 `/v1` 前缀。例外情况：
- GLM-4: `https://open.bigmodel.cn/api/paas/v4`（保留完整路径）
- 通义千问: `https://dashscope.aliyuncs.com/compatible-mode/v1`（保留完整路径）
- Azure: 使用独立路径格式

### Q: 如何指定端口？

```bash
PORT=9090 ./llm-tester
```

### Q: 配置存储在哪里？

`~/.llm_tester/configs.json`，仅当前用户可读写（权限 0600）。

### Q: 与其他 AI 工具冲突怎么办？

可能原因：
1. **代理冲突** — 确保代理地址配置正确
2. **端口冲突** — 使用 `PORT` 环境变量修改
3. **API Key 冲突** — 不同配置使用不同的 Key
