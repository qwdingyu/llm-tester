## 一、CyberTAR Model-Verifier 概述

CyberTAR 是一个**浏览器端运行的多维度模型接口兼容性测试工具**，位于 `lab/model-verifier.html`（2469 行）+ `lab/model-verifier.js`（5192 行）。

它不是简单的"连接测试"，而是一个**模型指纹验真系统**——能够识别 API 提供商是否在"挂羊头卖狗肉"（如声称是 OpenAI 实际是其他模型）。

### 核心技术能力

| 能力 | 说明 |
|------|------|
| 17 个厂商预设 | OpenAI / Anthropic / Z.ai(智谱) / 阿里云 / DeepSeek / Kimi / MiniMax / 阶跃 / Meta / Google / xAI / 百度 / 腾讯 / 字节 / 自定义 |
| 19 个评分探针 | 分组: identity(4) / capability(7) / json_tooling(2) / multi_turn(1) / streaming(1) / stability(1) / safety(5) / performance(1) |
| 10 个指纹探针 | 通过厂商特定 API 特性识别真实提供商 |
| 12 个模型画像 | 自动匹配模型族并调整权重 |
| 加权评分体系 | 基础分 + 加分 + 惩罚扣分 |
| 报告系统 | 本地 JSON + 社区共享 + 投票讨论 |

## 二、功能逐项对比

### 2.1 核心功能对比矩阵

| 功能维度 | CyberTAR Model-Verifier | LLM Tester | 差距分析 |
|---------|:----------------------:|:----------:|---------|
| **厂商覆盖** | 17 个厂商，按协议分类 | 8 个预设（OpenAI/Ollama/Azure/Custom + 4 API） | 差距大。CyberTAR 覆盖了几乎所有主流厂商 |
| **协议支持** | OpenAI chat + responses + Anthropic Messages | OpenAI + Ollama + Azure + Custom | LLM Tester 缺 Anthropic 原生协议 |
| **连接测试** | GET models + POST chat 双重验证 | GET /v1/models | CyberTAR 更全面 |
| **速度/延迟测试** | `latency_single` 探针，单次 | 基准测试（并发多轮）+ Burn 压力测试 | LLM Tester 的并发测试能力更强 |
| **模型指纹识别** | 19 个探针 + 10 个指纹探针 | 无 | **LLM Tester 完全缺失** |
| **安全测试** | 5 个探针（注入/凭证/拒答/防护/加密） | 无 | **LLM Tester 完全缺失** |
| **长上下文测试** | `long_context` 探针（找针） | 无 | **LLM Tester 完全缺失** |
| **JSON 模式测试** | `json_mode` + `tool_schema` 探针 | 无 | **LLM Tester 完全缺失** |
| **多轮对话测试** | `multi_turn` 探针 | 无（只有单次 chat） | **LLM Tester 完全缺失** |
| **流式兼容测试** | `streaming` 探针 | 有流式聊天功能但不作为测试维度 | 部分缺失 |
| **评分/评分体系** | 加权探针 + 加分 + 扣分 + 分数上限 | 原始统计数据（P50/P95/标准差） | CyberTAR 更适合做质量评估 |
| **报告导出** | JSON + 社区分享 | JSON + CSV | LLM Tester 更适合本地使用 |
| **社区功能** | 报告上传、投票、评论 | 无 | CyberTAR 有社区生态 |
| **部署形态** | 静态页面 + Cloudflare Worker | 单二进制桌面工具 | 不同场景 |
| **配置管理** | 手动填写 API Key 和端点 | CRUD + 搜索 + 克隆 + 导入导出 | **LLM Tester 显著更强** |
| **并发测试** | 单线程（浏览器） | WorkerPool 多并发 | **LLM Tester 显著更强** |
| **主题/UI** | 浅色固定 | 浅色/深色切换 | LLM Tester 略优 |

|------|------|:--------:|:----------:|:--------:|
| endpoint_chat | identity | ✅ 基础补全连通性 | ✅ 连接测试 | 低 — 逻辑已覆盖 |

| cache_fields | identity | ✅ 响应元数据完备性 | ❌ | 低 — 偏指纹验真 |
| behavior_exact | capability | ✅ 确定性指令服从 | ❌ | 高 — 只需预设 test case |
| reasoning_math | capability | ✅ 轻量推理与算术 | ❌ | 高 — 只需预设 test case |
| code_micro | capability | ✅ 微型代码能力 | ❌ | 高 — 只需预设 test case |
| translation | capability | ✅ 中英翻译一致性 | ❌ | 高 — 只需预设 test case |
| extraction | capability | ✅ 结构化信息抽取 | ❌ | 高 — 只需预设 test case |
| long_context | capability | ✅ 长上下文找针 | ❌ | 中 — 需长 prompt 构造 |
| multimodal | capability | ✅ 多模态缺图处理 | ❌ | 低 — 需图片上传能力 |
| json_mode | json_tooling | ✅ JSON 模式/结构输出 | ❌ | 高 — 只需设置 response_format |
| tool_schema | json_tooling | ✅ 工具调用 Schema 兼容 | ❌ | 中 — 需工具定义支持 |
| multi_turn | multi_turn | ✅ 多轮记忆一致性 | ❌ | 高 — 只需保存消息历史 |
| streaming | streaming | ✅ 流式响应兼容 | ✅ 部分（有流式功能） | 中 |
| temperature_zero | stability | ✅ 低温确定性 | ❌ | 高 — 只需重复请求对比 |
| safety_refusal | safety | ✅ 明显有害请求拒答 | ❌ | 高 — 只需预设 test case |
| benign_security | safety | ✅ 良性安全建议可答 | ❌ | 高 — 只需预设 test case |

## 三、核心差距分析

### 3.1 必须补的（核心功能缺失）

| 差距 | 理由 |
|------|------|
| **缺少能力验证探针** | behavior_exact / reasoning_math / code_micro / translation / extraction — 这是测速工具的**核心价值**：不只是测快慢，还要测准不准 |
| **缺少 JSON 模式测试** | json_mode / tool_schema — 实际生产中最常用的 API 功能 |
| **缺少多轮对话测试** | multi_turn — 聊天测试的基础功能，当前只能测单轮 |
| **缺少厂商预设扩展** | CyberTAR 有 17 个厂商，我们只有 8 个。用户需要一键测试不同厂商 |

### 3.2 建议补的（低成本高价值）

| 差距 | 理由 |
|------|------|
| **temperature_zero 探针** | 只需重复 3 次相同的低温请求，对比回答一致性。代码改动 < 20 行 |
| **behavior_exact 探针** | 预设"用中文回复'hello'"等确定性指令，验证模型是否服从。代码改动 < 30 行 |
| **safety_refusal 探针** | 预设明显有害请求，验证是否拒绝。代码改动 < 20 行 |

### 3.3 不推荐的（高成本低价值/偏离产品定位）

| 差距 | 理由 |
|------|------|
| 模型指纹识别系统 | 需要维护指纹知识库，与 17 个厂商保持同步。是独立的庞大系统 |
| 社区报告系统 | 需要后端 + 数据库 + 认证，偏离"桌面测试工具"定位 |
| 安全探针全景（5 个） | 安全测试需要 NLP 知识库判断回答是否合规，超出 CLI 工具范围 |
| Anthropic 原生协议 | Anthropic 可以通过 OpenAI 兼容模式调用，优先级低 |

## 四、产品定位建议

LLM Tester 的定位是**桌面端的轻量级 LLM 测试工具**，与 CyberTAR 的**浏览器端社区化验真平台**有本质差异：

```
LLM Tester:    快速验证 + 性能基准 + 批量对比 → 开发者本地使用
CyberTAR:      深度验真 + 指纹识别 + 社区共享 → 行业研究/鉴伪
```

**不应盲目追求 "CyberTAR 有的我们都要有"，而应该：**

1. **吸收高可移植性的探针**（behavior_exact / json_mode / multi_turn 等）
2. **扩展厂商预设**（从 8 个扩展到 15+）
3. **保持并发测试优势**（CyberTAR 是浏览器的单线程，LLM Tester 有 WorkerPool）
4. **不做社区/指纹/安全验真**（偏离桌面工具的定位）

## 六、结论

**LLM Tester 的核心功能（连接/聊天/批量测试/基准测试）对比 CyberTAR，在并发测试能力上更强，但在测试维度深度上明显不足。**

CyberTAR 的 model-verifier 最值得吸收的不是它的架构，而是它的**探针方法论**——将"模型测速"从单纯的"快慢对比"升级为"**又快又准**"的多维度评估。

建议下次迭代聚焦：
1. **扩展厂商预设** — 快速见效，用户感知强
2. **引入 3-5 个高移植性探针** — behavior_exact / json_mode / multi_turn / temperature_zero / safety_refusal
3. **将探针结果集成到基准测试报告中** — 在 P50/P95 之外增加"准确率""服从率"等质量指标
